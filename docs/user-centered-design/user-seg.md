# The First Step — Target Users, Concerns, and Minimum Features

## 1. Target Users

### Grade Level

The platform mainly targets:

* Freshmen
* Sophomores
* Juniors

These students are early enough in their college journey that they still need guidance, but they are also starting to think seriously about internships, career paths, and portfolio-building.

---

## 2. Target Majors and Roles

The platform is open to students from any major, as long as they want to gain project experience.

However, the main project roles are:

### Designer

Students interested in UI/UX, product design, branding, user research, or visual communication.

### Engineer

Students interested in software engineering, web development, data, AI, or technical product building.

### Marketer

Students interested in marketing, growth, content strategy, user acquisition, social media, or business development.

The platform should not only serve computer science students. It should help interdisciplinary teams form around real projects.

---

## 3. Target Situation

The target user is someone who wants to gain experience for an internship, job interview, or future career opportunity, but they are stuck because they do not know how to start.

They may have these problems:

* They want an internship but do not know what experience they need.
* They want to build a portfolio but do not know what project to choose.
* They do not know what career path fits them.
* They do not know what industry or field they are interested in.
* They do not know how to join a project.
* They do not know how to start a project by themselves.
* They do not know what roles are needed to complete a project.
* They do not know how to manage a project from beginning to end.

---

## 4. User Experience Level

The platform should support three types of users.

### Type 1: No Experience

These users have never joined a project before.
They may not know what a project looks like, what role they should take, or how people collaborate.

### Type 2: Weak or Unclear Experience

These users have joined something before, but the experience was not strong enough to use for internship applications or interviews.

For example:

* They joined a club project but did not have a clear role.
* They worked on a class project but it was not portfolio-ready.
* They started something but did not finish it.
* They have experience but cannot explain it clearly in interviews.

### Type 3: Career-Confused Users

These users do not even know how to get an internship or job.
They need guidance before they can even choose the right project.

They need help understanding:

* What career paths exist
* What skills each path requires
* What projects are useful for each path
* What experiences recruiters care about
* How to turn a project into a resume or interview story

---

# 5. Main User Concerns

The First Step should cover these major concerns.

## Concern 1: Career Direction

### User Question

“What field or industry am I interested in?”

### User Need

Users need help identifying what industries, roles, or fields they may want to explore.

### Minimum Feature

An onboarding survey that asks about:

* Interests
* Major
* Skills
* Target industries
* Target roles
* Career goals
* Past experience level

The platform should use this information to suggest possible career directions.

---

## Concern 2: Career Guidance

### User Question

“What kind of experience do I need to get an internship or job in that field?”

### User Need

Users need examples of what kinds of projects or experiences are helpful for different roles and industries.

### Minimum Feature

A career guidance page that explains:

* Example career paths
* Common entry-level roles
* Skills needed for each role
* Example projects that help users prepare for those roles
* How those projects connect to internship or job applications

Example:

If the user is interested in product design, the platform can explain that useful experiences may include UI redesign projects, user research projects, prototype projects, and design case studies.

If the user is interested in software engineering, the platform can explain that useful experiences may include web apps, APIs, data tools, AI apps, or open-source projects.

---

## Concern 3: Project Recommendation

### User Question

“What project should I choose?”

### User Need

Users need project suggestions based on their interests, career goals, and current skill level.

### Minimum Feature

A project recommendation system that suggests projects based on:

* User’s target role
* User’s target industry
* User’s current skill level
* User’s preferred project type
* User’s weekly time commitment

Each project card should show:

* Project title
* Target role
* Target industry
* Difficulty level
* Required roles
* Skills users will learn
* Final deliverable
* Estimated timeline

---

## Concern 4: Project Access

### User Question

“Where can I join or start a project?”

### User Need

Users need a place where they can either join existing projects or launch their own.

### Minimum Feature

The platform should provide two options:

1. Join an existing project
2. Start a new project

For existing projects, users should be able to browse project teams.

For new projects, users should be guided through a project setup process.

---

## Concern 5: Role Clarity

### User Question

“What roles do we need to finish this project?”

### User Need

Users need to understand what kind of teammates are required and what each person should do.

### Minimum Feature

Each project should clearly show required roles, such as:

* Designer
* Engineer
* Marketer
* Product Manager
* Business Strategist

Each role should include:

* Responsibilities
* Expected tasks
* Required baseline skills
* Weekly deliverables
* Final contribution to the project

---

## Concern 6: Team Formation

### User Question

“Who should I work with?”

### User Need

Users need to gather people who match their skill level, interests, and commitment.

### Minimum Feature

A team matching or team discovery feature that shows:

* Current team members
* Open roles
* Skill level
* School
* Major
* Weekly availability
* Project interest
* Verification status

Users should be able to request to join with low friction.

---

## Concern 7: Project Execution

### User Question

“How do we start and finish the project?”

### User Need

Even after users join a project, they still need structure. Without guidance, beginner teams may get confused, lose motivation, or fail to finish.

### Minimum Feature

A project workflow system with:

* Weekly milestones
* Step-by-step tasks
* Role-specific checklists
* Project timeline
* Final deliverable checklist
* Progress tracking
* Completion guide

The platform should assist the team from the beginning to the end of the project.

---

# 6. Minimum Feature Set for MVP

## Feature 1: Login and Profile

Users should be able to create an account and profile.

Profile should include:

* Name
* Icon/profile picture
* School
* Major
* Grade
* Skills
* Target role
* Target industry
* Experience level
* Weekly availability

---

## Feature 2: School or ID Verification

To protect users from scams, fake accounts, and catfishing, the platform should include some form of verification.

Possible verification methods:

* School email verification
* Student ID verification
* Manual approval
* LinkedIn or GitHub connection

For MVP, school email verification is probably the simplest.

---

## Feature 3: Onboarding Survey

The onboarding survey helps identify what the user wants.

It should ask:

* What grade are you in?
* What is your major?
* What role are you interested in?
* What industries are you interested in?
* Do you have project experience?
* What skills do you already have?
* What kind of internship or job do you want?
* How many hours per week can you commit?

---

## Feature 4: Career Guidance

The platform should show users what kinds of experiences are useful for different roles and industries.

For example:

* Designer → UI/UX case study, product redesign, user research project
* Engineer → web app, API project, AI tool, data dashboard
* Marketer → growth campaign, social media strategy, market research, product launch plan

---

## Feature 5: Project Recommendation

Based on the onboarding answers, the platform should recommend project ideas.

Each project should include:

* Title
* Description
* Target role
* Target industry
* Required roles
* Difficulty
* Timeline
* Skills learned
* Final deliverables

---

## Feature 6: Join or Start Project

Users should be able to either:

1. Join an existing project team
2. Start a new project and recruit teammates

This is important because some students may already know what they want to build, while others need to join something structured.

---

## Feature 7: Team Matching

Users should be able to find teammates based on:

* Role
* Skill level
* School
* Major
* Availability
* Project interest
* Commitment level

This helps reduce mismatch and imposter syndrome.

---

## Feature 8: Messaging or Communication Tool

Teams need a way to communicate.

For MVP, this can be simple:

* Direct message
* Team chat
* Project comment section
* Discord or Slack link integration

The simplest MVP version may not need a full chat system. It could allow users to connect through email, Discord, or external links first.

---

## Feature 9: Project Workflow

Once a team starts, the platform should guide them through the project.

The workflow should include:

* Week 1: Team setup and project planning
* Week 2: Research and basic prototype
* Week 3: Core development or execution
* Week 4: Testing, improvement, and final output
* Week 5: Portfolio packaging and presentation

---

# 7. MVP User Flow

## Step 1: User signs up

The user creates an account and verifies their school email.

## Step 2: User creates profile

The user adds their school, major, grade, skills, target role, and availability.

## Step 3: User completes onboarding survey

The platform learns the user’s interests, career goals, and experience level.

## Step 4: Platform suggests career directions

The platform shows possible career paths, industries, and roles based on the user’s answers.

## Step 5: Platform recommends projects

The platform recommends structured project ideas that match the user’s interests and career goals.

## Step 6: User chooses to join or start a project

The user can either join an existing team or launch a new team.

## Step 7: User finds teammates

The user views team members, roles, skill levels, and availability.

## Step 8: Team follows project workflow

The team uses milestones, role checklists, and weekly tasks to complete the project.

## Step 9: Team finishes project

The team completes final deliverables that can be used for resumes, portfolios, LinkedIn, or interviews.

---

# 8. Simplified Product Concept

The First Step is a beginner-friendly project platform that helps students discover career paths, choose relevant projects, form teams, and complete portfolio-ready work.

It is designed for freshmen, sophomores, and juniors who want internship or job experience but do not know where to start.

The platform guides users through the full journey:

1. Identify interests and career direction
2. Understand what experience is needed
3. Choose a useful project
4. Find teammates
5. Follow a structured workflow
6. Complete a portfolio-ready outcome

# First Step Prototype Scope

Avoid scope creep by figuring out what our focus for the design is. The first interactive version of the application should be fixed to a minimal and testable baseline.

### Primary Prototype Audience

* **Target Cohort:** Motivated freshman, sophomore, and junior students with little to no practical project experience who want internship-ready proof.

* **Launch-Priority Roles:** Engineer (Software/Web/Data), Designer (UI/UX/Research), and Business/Product (Strategy/Marketing). 


# Prototype Must-Have Scope

This is the version for testing the idea with users, maybe even manually behind the scenes.

---

## Core Components

### 1. Onboarding & Recommendations
* **Simple Onboarding Survey:** Collect major, year, target role, interests, experience level, and weekly availability.
* **Career / Role Recommendation Result:** Show the user 1-3 suggested directions: Engineer, Designer, or Business/Product/Marketing.

### 2. Project Exploration
* **Project Track Cards:** A small curated set of beginner projects, maybe 6-9 total:
  * Web app / campus tool
  * Data dashboard
  * UX redesign
  * Product discovery sprint
  * Marketing/growth campaign
  * Consulting-style strategy project

* **Project Detail Page:** Each project explicitly includes:
  * Goal
  * Required roles
  * Difficulty
  * Timeline
  * Final output
  * Weekly milestones
  
* **Role Guide:** Explain what each role does in the project for:
  * Engineer
  * Designer
  * Business/Product/Marketing

### 3. Team Formation & Execution
* **Join Interest / Application Form:** Not real matching yet. Just an “I’m interested” CTA capturing chosen role, availability, and contact info.
* **Manual Team Formation:** Match students manually using a spreadsheet or admin background process.
* **Weekly Milestone Checklist:** A static or semi-static checklist for 4-5 weeks. The key is structure, not automation.

### 4. Project Closure
* **Final Output Guide:** Help users turn the project into a resume bullet, portfolio summary, GitHub README, case study, or deck outline.

---

## Primary Objective
* **Prototype Goal:** Validate whether students understand the value of the platform and would actually commit.
