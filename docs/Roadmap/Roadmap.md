# Roadmap

## Phase 1: Problem Validation

**Goal:**
Understand whether students actually need this platform.

**Actions:**

* Interview students and get their pain point
* Ask why they struggle to find internship or jobs
* Identify what support beginners need
* Compare the problem with LinkedIn, Discord, hackathons, and student clubs

**Outcome:**
Clear understanding of the target users and their biggest pain points.

---

## Phase 2: Define Target Users

**Goal:**
Decide who we are building for first.

**Target Users:**

* College students with little project experience
* Make Persona
* Students who could not get internships
* Students interested in business, product, design, tech, or startups
* Motivated beginners who want to grow but do not know where to start

**Outcome:**
A clear first user group and core use case.

---

## Phase 3: Build the MVP

**Goal:**
Create the simplest version of the platform to test the idea.

**Core Features:**

* User profile
* Beginner-friendly project list
* Team matching
* Role guide
* Weekly milestones
* Final portfolio/resume output

**Outcome:**
A simple prototype that users can try without building a full app yet.

---

## Phase 4: Run the First Pilot

**Goal:**
Test the platform with real users.

**Pilot Plan:**

* Recruit 10–30 students
* Form teams of 3–5 people
* Run a 3–6 week project cycle
* Let each team complete one final project

**Outcome:**
Real feedback on whether the platform helps beginners gain project experience.

---

## Phase 5: Improve the Product

**Goal:**
Improve the platform based on user feedback.

**Focus Areas:**

* Onboarding
* Team matching
* Project difficulty
* Role explanation
* Weekly milestone structure
* Final output quality

**Outcome:**
A stronger version of the platform based on real user behavior.
